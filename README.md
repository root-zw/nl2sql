## 系统概述

### 什么是NL2SQL智能问数系统？

NL2SQL智能问数系统是一个企业级的自然语言转SQL查询平台，允许用户使用自然语言提问，系统自动理解语义、生成SQL并执行查询，最终以表格、图表等形式展示结果。

### 核心特性

| 特性 | 描述 |
|------|------|
| 🗣️ **自然语言理解** | 支持复杂的中文自然语言查询，自动识别时间、地点、指标等语义要素 |
| 🔄 **混合架构** | 支持IR（中间表示）流程和直接SQL生成两种模式，自动选择最优路径 |
| 📊 **多数据源支持** | 支持SQL Server、MySQL、PostgreSQL等多种数据库 |
| 🔍 **智能检索** | 基于向量检索的表/字段/枚举值智能匹配，支持RRF融合和Reranker精排 |
| 🛡️ **企业级权限** | 行级权限(RLS)、列级权限、多租户隔离 |
| 📈 **可视化展示** | 支持表格、柱状图、折线图、饼图等多种可视化方式 |
| 💬 **流式响应** | WebSocket实时流式输出，支持进度展示和叙述生成 |
| 🔐 **多认证方式** | 支持本地认证、OIDC/SSO、LDAP、API网关等多种认证方式 |

## 核心模块详解

### 1. 查询路由层 (`server/api/query/`)

查询路由层是系统的入口，负责接收用户查询并分发到合适的处理链路。

```
┌────────────────────────────────────────────────────────────┐
│                    WebSocket /api/query/stream              │
├────────────────────────────────────────────────────────────┤
│  1. 认证校验 (JWT Token)                                    │
│  2. 连接检测 (自动/手动选择数据库连接)                       │
│  3. 缓存检查 (Redis)                                        │
│  4. 路由决策:                                               │
│     ├─ LLM表选择链路 (推荐)                                 │
│     ├─ 向量检索链路 (备选)                                  │
│     └─ 直接SQL链路 (复杂查询)                               │
│  5. 权限过滤 (RLS注入)                                      │
│  6. SQL执行                                                 │
│  7. 结果格式化与流式响应                                    │
└────────────────────────────────────────────────────────────┘
```

**核心文件：**
- `routes.py` - 主路由，包含WebSocket流式查询和REST接口
- `stream_emitter.py` - 流式事件发射器
- `table_selection.py` - LLM表选择逻辑
- `permission_checker.py` - 权限校验
- `visualization.py` - 可视化推荐

**核心组件：**

| 文件 | 功能 |
|------|------|
| `parser.py` | NL2IR主解析器，调用LLM生成IR |
| `hierarchical_retriever.py` | 层次化检索器，业务域→表→字段 |
| `enum_value_retriever.py` | 枚举值检索器，精确匹配字段值 |
| `few_shot_retriever.py` | Few-Shot检索器，相似问答召回 |

**IR数据结构 (`server/models/ir.py`)：**

```python
class IntermediateRepresentation(BaseModel):
    """中间表示 - 描述查询意图的结构化对象"""
    
    # 查询类型
    query_type: str  # aggregation/detail/window_detail/duplicate_detection
    
    # 核心要素
    metrics: List[str]      # 度量字段ID列表
    dimensions: List[str]   # 维度字段ID列表
    filters: List[FilterCondition]  # 过滤条件
    
    # 时间范围
    time: Optional[TimeRange]
    
    # 排序与限制
    order_by: Optional[List[OrderByClause]]
    limit: Optional[int]
    
    # 高级特性
    with_total: bool = False  # 是否包含合计行
    comparison_type: Optional[str]  # 同比/环比类型
    join_strategy: Optional[str]  # JOIN策略
    
    # 复杂查询标记
    requires_direct_sql: bool = False
    is_too_complex: bool = False
```

```
│     └──▶ RRF融合 ──▶ Reranker精排 ──▶ 最终结果             │
│                                                              │
│  RRF公式: score = Σ 1/(k + rank_i) × weight_i               │
```

### 复杂查询流程（DAG模式）

```
用户输入: "2024年各区成交宗数最多的是哪个区？成交了多少宗？"
                │
                ▼
┌───────────────────────────────────────────────────────────────────────────┐
│ 复杂度判断                                                                 │
│ ├── LLM识别: is_too_complex = true                                        │
│ └── 触发CoT规划                                                            │
└───────────────────────────────────────────────────────────────────────────┘
                │
                ▼
┌───────────────────────────────────────────────────────────────────────────┐
│ CoT规划器                                                                  │
│ ├── 分析问题，拆解为多步骤                                                 │
│ └── 生成DAG计划:                                                           │
│     {                                                                      │
│       "should_split": true,                                                │
│       "steps": [                                                           │
│         {"id": "step1", "description": "统计各区成交宗数", "deps": []},    │
│         {"id": "step2", "description": "找最大值", "deps": ["step1"]}      │
│       ]                                                                    │
│     }                                                                      │
└───────────────────────────────────────────────────────────────────────────┘
                │
                ▼
┌───────────────────────────────────────────────────────────────────────────┐
│ DAG执行器                                                                  │
│ ┌─────────────────────────────────────────────────────────────────────┐   │
│ │  Step1: 统计各区宗数                                                 │   │
│ │  ├── NL2IR: 生成聚合IR                                               │   │
│ │  ├── 编译: SELECT region, COUNT(*) FROM ... GROUP BY region          │   │
│ │  └── 执行: 返回各区宗数列表                                          │   │
│ └─────────────────────────────────────────────────────────────────────┘   │
│                            │                                               │
│                            ▼                                               │
│ ┌─────────────────────────────────────────────────────────────────────┐   │
│ │  Step2: 找最大值                                                     │   │
│ │  ├── 接收Step1结果作为上下文                                         │   │
│ │  ├── NL2IR: 生成排序+限制IR                                          │   │
│ │  ├── 编译: SELECT TOP 1 ... ORDER BY count DESC                      │   │
│ │  └── 执行: 返回最大值行                                              │   │
│ └─────────────────────────────────────────────────────────────────────┘   │
└───────────────────────────────────────────────────────────────────────────┘
                │
                ▼
┌───────────────────────────────────────────────────────────────────────────┐
│ 结果聚合                                                                   │
│ └── 返回: "江汉区, 成交宗数: 150宗"                                        │
└───────────────────────────────────────────────────────────────────────────┘
```


数据库初始化脚本会在Docker启动时自动执行，或手动执行：


```bash
conda activate text2sql
uvicorn server.main:app --host 0.0.0.0 --port 8890 --reload
```

```bash
cd frontend
npm install
npm run dev
```

- 前端查询界面: http://localhost:3000
- 管理后台: http://localhost:3000/admin
- API文档: http://localhost:8890/docs

## 开发指南

### 代码规范

- **Python**: PEP8, 4空格缩进, 类型注解
- **前端**: Vue 3 Composition API, ESLint

### 添加新的数据库支持

1. 在 `server/compiler/` 下创建新的方言处理器
2. 在 `SQLCompiler._get_dialect_handler()` 注册
3. 在 `QueryExecutor` 添加连接支持

### 添加新的LLM提供商

1. 确保API兼容OpenAI接口
2. 在 `.env` 配置相应的 `*_BASE_URL` 和 `*_API_KEY`
3. 如需特殊处理，扩展 `LLMClient`

所有Prompt模板位于 `prompts/` 目录，可通过环境变量指定自定义路径：

## 常见问题

### Q: 查询超时怎么办？

A: 调整 `QUERY_TIMEOUT_SECONDS` 配置，或优化查询条件减少数据量。

### Q: LLM返回结果不准确？

A: 
1. 检查表/字段描述是否清晰
2. 添加Few-Shot样本
3. 调整Prompt模板
4. 尝试更强的LLM模型

### Q: Milvus同步失败？

A: 
1. 检查Milvus服务状态
2. 检查Embedding服务是否正常
3. 查看同步日志定位问题
